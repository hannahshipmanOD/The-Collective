# Meeting Capture Data Model

## Participant lookup

When participants are mentioned, cross-reference with rosters:
- `config/xa-roster.json` — XA team
- `config/tc-roster.json` — TC team

## Common meeting types at Opendoor

| Meeting type | Key things to capture |
|-------------|----------------------|
| Team standup | Blockers, metrics anomalies, celebrations |
| 1:1 | Decisions, career items, asks |
| Incident review | Root cause, action items, timeline |
| Sprint planning | Commitments, owners, deadlines |
| Cross-functional sync | Decisions, who owns what, follow-up |
| Leadership review | Strategic decisions, resource asks |

## Action item taxonomy

| Priority | Definition |
|----------|------------|
| P0 | Blocking something or someone right now |
| P1 | Due this week |
| P2 | Due this sprint/cycle |
| P3 | Good to have, no hard deadline |
